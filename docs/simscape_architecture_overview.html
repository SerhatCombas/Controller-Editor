Phase 1 to Phase 2 Backbone

Lightweight Simscape-Style Engine for Predefined Templates First, Visual Builder Later

The key architectural decision is to build the simulation kernel now as a port-based graph engine, while the Phase 1 UI exposes only beginner-friendly predefined templates. The UI never owns equations. Templates only instantiate components and connections. The graph, symbolic, and solver layers derive the system model automatically.

System Flow

Template or Editor

Creates components, ports, probes, and connections.

System Graph

Stores topology, nodes, domains, and connectivity.

Equation Builder

Combines constitutive equations with node laws.

Solver + Analysis

Builds ODE/state-space and runs simulation or linearization.

UI + Plots

Displays probes, matrices, warnings, and analysis results.

Core/Base

Primitive Modeling Objects

  • Domain: names the physical domain and across/through semantics.
  • Port: typed terminal with a node reference and symbolic variables.
  • Node: enforces across-equality and through-sum-zero.
  • Connection: binds ports and lets graph analysis merge nodes.
  • BaseComponent: reusable physical element contract.
Graph

Topology and Validation

  • Stores components and their port-level connectivity.
  • Rejects domain mismatch and invalid source conflicts.
  • Detects isolated subsystems and missing references.
  • Classifies model as mechanical, electrical, or mixed.
  • Publishes structural metadata to the UI.
Symbolic + Solver

Automatic Equation Pipeline

  • Constitutive equations come from components.
  • Node equations come from graph assembly.
  • Builders create ODE and state-space forms.
  • Transfer functions only when linear and valid.
  • Nonlinear systems go through linearization first.

Quarter-Car Template Example
def build_quarter_car_template(): graph = SystemGraph() body = Mass("body_mass", m=300.0) wheel = Mass("wheel_mass", m=40.0) suspension = Spring("suspension_k", k=15000.0) damper = Damper("suspension_c", d=1200.0) tire = Spring("tire_k", k=180000.0) road = RandomRoad("road_source", ...) ground = MechanicalGround("ground") graph.add_component(body) graph.add_component(wheel) graph.add_component(suspension) graph.add_component(damper) graph.add_component(tire) graph.add_component(road) graph.add_component(ground) graph.connect(body.port, suspension.port_a) graph.connect(body.port, damper.port_a) graph.connect(suspension.port_b, wheel.port) graph.connect(damper.port_b, wheel.port) graph.connect(wheel.port, tire.port_a) graph.connect(tire.port_b, road.port) graph.connect(road.reference_port, ground.port) graph.attach_probe(DisplacementProbe("body_disp", body)) graph.attach_probe(DisplacementProbe("wheel_disp", wheel)) graph.attach_probe(AccelerationProbe("body_acc", body)) graph.attach_probe(ForceProbe("tire_force", tire)) return TemplateDefinition( graph=graph, default_input="road_source", default_output="body_disp", suggested_plots=["time", "step", "bode", "pole_zero"] )
What Is Explicitly Not Hardcoded
  • Quarter-car equations are not handwritten in the solver.
  • Input/output choices are not fixed in the template view.
  • Plots are driven by the selected probe and selected source.
  • Future visual editor uses the same graph object, not a parallel scene model.

Templates are convenience front-ends, not a second simulation architecture.

Phase Plan
Phase 1

Implement domains, ports, graph validation, a minimal symbolic builder for linear translational and electrical components, predefined templates, probe-driven outputs, and the PySide6 template UI.

Phase 2

Add the graphical editor as a view/controller over the same graph model: drag components, create node connections, snap wires, edit properties, run validation, and simulate immediately.

How the Future Editor Reuses the Backend
  • Canvas items map 1:1 to BaseComponent instances.
  • Visual ports map 1:1 to Port objects.
  • Wires create Connection objects and trigger graph rebuild.
  • Validation errors come from graph validators and show in UI badges.
  • Equation, state-space, and plots refresh from the same services used by templates.
